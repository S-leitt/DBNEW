### 数据库连接方案

根据对项目代码和配置的分析，您的系统已经配置了三个数据库（MySQL、Oracle、SQL Server），并且实现了自动连接机制。以下是连接三个数据库的详细方案：

#### 1. 检查当前配置

**配置文件位置**：`f:\桌面\database\src\main\resources\application.yml`

**当前配置状态**：
- ✅ MySQL 配置：已配置，端口 3306，数据库 exam_paper_db
- ✅ Oracle 配置：已配置，端口 1521，服务名 ORCLPDB
- ✅ SQL Server 配置：已配置，端口 1434，数据库 exam_paper_db

#### 2. 启动数据库同步系统

**启动命令**：
```powershell
& "D:\Apache\apache-maven-3.9.12\bin\mvn.cmd" spring-boot:run
```

**预期结果**：
- 系统启动时会自动初始化所有三个数据库连接
- 日志中会显示每个数据库的连接状态
- 成功连接后会显示 "数据源 xxx 连接测试成功"

#### 3. 验证连接状态

**验证方式**：
1. 查看启动日志，确认三个数据库都显示连接成功
2. 访问系统前端：`http://localhost:8080/sync`
3. 在前端界面中查看数据库连接状态

#### 4. 可能的问题排查

**MySQL 连接失败**：
- 检查 MySQL 服务是否启动
- 验证用户名和密码是否正确
- 确认端口 3306 未被占用

**Oracle 连接失败**：
- 检查 OracleServiceORCL 服务是否启动
- 确认监听器（Lsnrctl status）是否正常
- 验证 Oracle 用户和密码是否正确

**SQL Server 连接失败**：
- 检查 SQL Server 服务是否启动
- 确认是否启用了 SQL Server 身份验证
- 验证 sa 用户密码是否正确
- 确认端口 1434 配置正确

#### 5. 连接机制说明

系统采用以下机制自动连接三个数据库：
1. `DatabaseConfigLoader` 类在启动时读取 `application.yml` 中的所有数据库配置
2. 为每个数据库创建 `DatabaseConfig` 对象
3. 调用 `DatabaseConnectionFactory.initDataSource()` 初始化数据源
4. 使用 HikariCP 连接池管理数据库连接
5. 启动时自动测试每个数据库连接

**核心实现类**：
- `DatabaseConnectionFactory`：负责创建和管理数据源
- `DatabaseConfigLoader`：负责加载配置并初始化数据源
- `DatabaseConfig`：存储数据库连接配置信息

#### 6. 后续操作

连接成功后，您可以：
- 在前端界面查看同步任务状态
- 测试数据同步功能
- 监控数据库连接池状态
- 查看同步日志和冲突记录

**注意事项**：
- 确保所有数据库服务都已启动
- 验证防火墙设置，确保数据库端口可访问
- 首次启动可能需要较长时间初始化连接
- 如果某个数据库连接失败，系统会继续启动，但该数据库的同步功能将不可用
